# 公众号大板块信息图生成技能

## 技能定位

- **触发时机**：最终文章定稿后（交付文件夹 `<文章标题>/<文章标题>.md` 已生成）。
- **做什么**：解析文章中每个 `## ` 二级标题，为每个章节生成一张**手绘笔记风格**信息图（16:9，白底），存入 `<文章目录>/image/`，并把 `![章节信息图](image/xxx.png)` 插入到该章节正文的末尾（原地改写交付稿）。
- **密钥**：脚本自动从环境变量 `GEMINI_KEY`（或 `.claude/settings.local.json` 的 `env.GEMINI_KEY`）读取，无需手动传。

## 输入

- `$ARGUMENTS` = 交付稿路径，例如 `AI时代，普通人到底该不该焦虑被淘汰？/AI时代，普通人到底该不该焦虑被淘汰？.md`。
- 未提供时：读 `.claude/active-article`，定位其**交付稿**（项目根 `<标题>/<标题>.md`）；若不存在则询问用户。

## 工作流程

### 第一步 — 定位文章并说明计划
- 确定交付稿绝对路径（按上面"输入"规则）。
- 向用户说明：即将为该文章的每个 `##` 章节生成信息图，图片存 `<交付目录>/image/`，并原地把图片链接插入各章节末尾；**会改写交付稿**（如需保留无图原版，请先自行复制一份）。

### 第二步 — 预检（dry-run，必做）
先跑一次预检，确认能正确识别章节：
```bash
python ".claude/skills/infographic_generation/infographic_generator.py" --article "<交付稿绝对路径>" --dry-run
```
- 若输出"未发现 ## 二级标题"：说明该文章用 `---` 分隔、没有 `##`。**停下告知用户**：需要先为各章节补 `## ` 二级标题再跑（本技能严格按 `##` 切分）。
- 若列出了若干章节：把章节清单给用户过目，确认后进入第三步。

### 第三步 — 正式生成并插入
```bash
python ".claude/skills/infographic_generation/infographic_generator.py" --article "<交付稿绝对路径>"
```
- 脚本会逐章生图、存入 `<交付目录>/image/`，并原地把 `![<标题>信息图](image/xxx.png)` 插入各章节末尾。
- 默认图片目录即 `<交付目录>/image/`（符合"每篇文章文件夹下的 image/"）；如需他用可加 `--output-dir "<目录>"`。

### 第四步 — 汇报结果
- 告知用户：成功几张 / 共几章、图片目录、被改写的交付稿路径。
- 列出每章对应的信息图文件名，方便用户核对。
- 提示：本技能**不做视觉质检**（按约定）；如某张不满意，可单独用"单图模式"重生成后手动替换：
  ```bash
  python ".claude/skills/infographic_generation/infographic_generator.py" "<该章节的生图提示词>" "<交付目录>/image"
  ```

## 执行约束

- **严格按 `##` 切分**：只认 `## ` 二级标题，不认 `#`（文章大标题）与 `###`。文章无 `##` 时不生图、不改稿，告知用户先补标题。
- **原地改写交付稿**：会修改 `<标题>/<标题>.md`；执行前已用 dry-run 预检并告知用户。
- **图片位置**：固定 `<交付目录>/image/`（相对链接写作 `image/xxx.png`）。
- **不质检**：生成即插入，由用户事后看图决定是否重跑某章。
- **联网与密钥**：脚本经 Gemini 生图，密钥从 `GEMINI_KEY` 读取；主程序不需自己联网。
- **不捏造内容**：信息图基于章节真实正文生成；脚本只把章节原文作为生图输入。
